# Weekly Production Schedule — history-production (long-form)

**Основной ритм:** 1 длинное видео в неделю, публикация — **вторник**.

> Шортсы — отдельный трек, спецификация в ветке `shorts`, файл `docs/WEEKLY_PRODUCTION_SCHEDULE.md`.

Якорь — вторник (публикация long-form). Всё остальное считается от него назад.

---

## Long-form pipeline (Дима + Настя + Lead)

| День | Кто | Что |
|---|---|---|
| **Ср** | Claude (Lead) + Codex + Gemini | Генерация сценария |
| **Чт** | Ярослав | Утверждение сценария. **Старт генерации футажей** (Дима — основной, Артём — помощь) |
| **Пт** | Дима | Монтаж первой версии. Финальный рендер — **вечер пятницы**, кладётся в Production forum / репо |
| **Пт** | Настя | Несколько вариантов превью, готовы к субботе |
| **Пт (ночь) / Сб (утро)** | Ярослав + Codex (GPT-5.4 Vision) | Просмотр рендера, базовые правки (чёрные кадры, артефакты, тайминг) через нативный video-input |
| **Сб** | Команда (встреча по продакшену) | Даём Диме правки по монтажу, Насте — правки по превью |
| **Вс** | Дима | Реализация правок по видео |
| **Вс** | Настя | Реализация правок по превью |
| **Пн** | Дима + Настя | **Публикация-preflight** (видео + превью), с учётом всех правок |
| **Пн** | Команда | Финальный просмотр, ловим недочёты |
| **Вт** | — | **Публикация вживую** (запас ~1 день на экстренные правки) |

> Буфер в 1 день между понедельником и вторником — страховка на случай финальных косяков.

---

## Summary (быстрый взгляд)

```
Вт  → публикация long-form ← якорь
Ср  → сценарий (Lead)
Чт  → утверждение сценария + старт футажей (Дима, Артём)
Пт  → монтаж v1 (Дима) | превью v1 (Настя)
      вечер: рендер в репо → Ярослав + Codex Vision ревью
Сб  → встреча, правки монтажа и превью
Вс  → правки (Дима, Настя)
Пн  → финальная сборка, preflight
Вт  → публикация ← якорь
```

---

## Роли в ревью

- **Ярослав** — финальный review всего, базовые video-правки через Codex (GPT-5.4 Vision)
- **Lead (Claude)** — оркестрация, сценарий, монтаж-план, координация треков
- **Codex** — video review через Vision, prompt-engineering для футажей
- **Gemini** — research, proofreading (параллельно с Codex), competitor analysis
- **Дима** — монтаж long-form, основная генерация футажей, публикация long-form
- **Артём** — помощь с футажами long-form (основной трек — шортсы, ветка `shorts`)
- **Настя** — превью, проверка монтажа, аналитика
