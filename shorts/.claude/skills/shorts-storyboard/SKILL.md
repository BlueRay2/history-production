# Shorts Storyboard — Визуальный сценарий для YouTube Short

## Назначение

Генерация полного визуального сторибоарда для YouTube Short на основе секции 9 файла `docs/ref/NICHE_STYLE_GUIDE.md`. Результат — набор сцен, каждая из которых содержит:

1. **Промпт для генерации начального кадра** (статичная картинка в Nano Banana или GPT)
2. **Промпт для генерации конечного кадра** (статичная картинка — становится начальным кадром следующей сцены)
3. **Промпт для Kling AI перехода** (I2V Start/End Frame — реалистичный переход между двумя картинками)

**Формат Shorts:** чисто визуальный сторителлинг с музыкой/звуком. Без озвучки, без текстовых оверлеев — только видео + аудио.

**Принцип:** Shorts строятся на реалистичных переходах через Kling AI. Каждая сцена — это анимированный переход между двумя сгенерированными картинками. Конечный кадр сцены N = начальный кадр сцены N+1 (цепочка).

## Предусловие

- Файл `docs/ref/NICHE_STYLE_GUIDE.md` должен существовать.
- Секция 9 (Тема текущего видео) должна быть заполнена — без `{{...}}` плейсхолдеров.
- Если секция 9 не заполнена — направить пользователя на `/setup-video`.

## Алгоритм

### Шаг 1 — Чтение и валидация

- READ `docs/ref/NICHE_STYLE_GUIDE.md`.
- Проверить, что секция 9 заполнена (нет `{{...}}`).
- Извлечь: тему, визуальную концепцию, референс-пример, настроение, обязательные элементы, исключения.
- Извлечь из секций 1-8 и 10: стиль канала, целевую аудиторию, визуальный язык.

### Шаг 2 — Анализ визуальной концепции и референса

На основе секции 9 определить:

- **Визуальная тема** — что зритель видит на экране.
- **Референс-пример** — проанализировать описание референсного видео (если указано): какие переходы нравятся, стиль, ритм, настроение. Использовать как ориентир для типов переходов.
- **Настроение / эмоция** — какое чувство должен вызвать Short.
- **Музыкальное настроение** — темп, жанр, энергия.
- **Паттерн сторителлинга** — выбрать один из паттернов (см. ниже).
- **Стратегия зацикливания** — как последний кадр переходит обратно в первый.

### Шаг 3 — Проектирование визуальной арки (ключевые кадры)

Спланировать последовательность **ключевых кадров** (keyframes):

- Типичный Short: **4-8 сцен** (переходов), **5-10 секунд** каждый переход.
- Это означает **5-9 ключевых кадров** (на 1 больше, чем сцен).
- Суммарная длительность: **≤60 секунд**.
- Первый кадр — **хук** (визуально цепляющий, необычный).
- Последний кадр — визуально связан с первым для **зацикливания**.
- Каждый кадр должен быть достаточно отличным от предыдущего, чтобы переход был интересным.

### Шаг 4 — Генерация промптов для картинок (Nano Banana / GPT)

Для каждого ключевого кадра сгенерировать промпт для генерации статичной картинки:

- **Генератор:** Nano Banana или GPT (image generation).
- **Формат:** 9:16 вертикальный (1080×1920).
- **Длина промпта:** 40-100 слов — детальное описание сцены.
- **Что включать:**
  - Субъект и его поза/состояние
  - Окружение и фон
  - Освещение (конкретные источники света, не абстрактные)
  - Цветовая палитра (единая по всем кадрам)
  - Композиция в вертикальном кадре (субъект по центру)
  - Стилевой суффикс (единый для всех кадров батча)
- **Визуальная целостность:** все кадры должны выглядеть как часть одного видео — единый стиль, палитра, освещение.

### Шаг 5 — Генерация промптов для переходов (Kling AI)

Для каждой пары последовательных кадров сгенерировать промпт для Kling AI:

- **Режим:** I2V с Start/End Frame (Start Frame = картинка начала сцены, End Frame = картинка конца сцены).
- **Длина промпта:** 20-40 слов — описание только движения и трансформации.
- **Что включать:**
  - Тип трансформации / перехода (морфинг, движение камеры, изменение сцены, зум и т.д.)
  - Направление и скорость движения
  - Что происходит между двумя кадрами
- **Чего НЕ включать:**
  - Описание самих кадров (Kling видит их из загруженных картинок)
  - Стилевые суффиксы (стиль задан картинками)
- **КРИТИЧНО — терминология Kling:** Kling инвертирует pan/tilt:
  - Kling `pan` = вертикальное движение (стандартный tilt).
  - Kling `tilt` = горизонтальное движение (стандартный pan).
- **Модель Kling:** 2.5 Turbo для простых переходов, 3.0 для сложных трансформаций.
- **Кредиты:** расчёт для каждого перехода.

### Шаг 6 — Визуальная целостность

Обеспечить единство по всем кадрам и переходам:

- **Стилевая линия:** одинаковый стилевой суффикс во всех промптах для картинок.
- **Цветовая палитра:** 2-3 доминантных цвета, соблюдать во всех кадрах.
- **Направление света:** выбрать одно и выдерживать.
- **Цепочка кадров:** конечный кадр сцены N = начальный кадр сцены N+1 (одна и та же картинка).
- **Субъект по центру:** в 9:16 всё важное в центре кадра.

### Шаг 7 — Планирование петли (Loop)

Спроектировать как последний кадр визуально переходит обратно в первый:

- Совпадение по цветовой температуре и общей композиции.
- Последний кадр ≈ первый кадр по настроению и расположению элементов.
- Последний переход (сцена N → кадр 1) замыкает цикл.

### Шаг 8 — Аудио-заметки

Для каждой сцены (перехода) добавить заметку по аудио:

- Что должно звучать (музыка, звуковой эффект, амбиент).
- Для переходов, где нужен нативный звук — отметить Kling 2.6.
- Для остального — заметки для пост-продакшена.

### Шаг 9 — Расчёт стоимости

Подсчитать общую стоимость:

**Kling AI (переходы):**
| Операция | Кредиты |
|----------|---------|
| I2V 5s Professional (Start/End Frame) | 35 |
| I2V 10s Professional (Start/End Frame) | 70 |
| I2V + Audio (Kling 2.6) | 50-200 |

**Генерация картинок (Nano Banana / GPT):**
- Стоимость зависит от выбранного генератора (указать примерную).

Добавить 20% overhead на неудачные генерации и итерации.

## Паттерны сторителлинга для Shorts

Выбрать один паттерн, наиболее подходящий теме из секции 9:

| Паттерн | Описание | Арка |
|---------|----------|------|
| **Трансформация** | До/после | A -> процесс -> B |
| **Удовлетворяющая петля** | Конец = начало | A -> ... -> A |
| **Эскалация** | Простое -> сложное | 1 -> 2 -> ... -> N (нарастание) |
| **Контраст** | A vs B | Чередование двух противоположностей |
| **Путешествие** | Визуальное перемещение из A в B | Старт -> путь -> финиш |
| **Раскрытие** | Загадка -> ответ | Интрига -> подсказки -> reveal |
| **Процесс** | Пошаговое создание | Этап 1 -> 2 -> ... -> результат |

## Типы переходов между кадрами

Для каждой пары кадров выбрать тип перехода, который Kling AI выполнит:

| Тип перехода | Описание | Когда использовать |
|-------------|----------|-------------------|
| **Морфинг** | Один объект/сцена плавно превращается в другой | Трансформации, метаморфозы |
| **Движение камеры** | Камера движется, открывая новую сцену | Путешествия, раскрытия |
| **Зум-переход** | Зум в деталь → деталь становится новой сценой | Эскалация, углубление |
| **Смена времени** | Та же сцена, но другое время/сезон/эпоха | Контраст, трансформация |
| **Растворение** | Элементы одной сцены растворяются, формируя новую | Мягкие переходы, настроение |
| **Движение объекта** | Объект движется и "уносит" камеру в новую сцену | Динамичные переходы |
| **Поворот/вращение** | Сцена вращается, открывая новый ракурс/мир | Контраст, сюрприз |

## Ключевые ограничения

1. **Хук в первом кадре** — первый ключевой кадр должен быть визуально сильным: необычный ракурс, яркий цвет, интригующая сцена.
2. **Реалистичные переходы** — каждая сцена = Kling AI переход между двумя картинками. Переход должен выглядеть естественно и кинематографично.
3. **Визуальная консистентность** — единый стилевой суффикс, цветовая палитра и освещение во всех промптах для картинок.
4. **Цепочка кадров** — конечный кадр сцены N = начальный кадр сцены N+1 (одна картинка используется дважды).
5. **≤60 секунд суммарно** — жёсткий лимит YouTube Shorts.
6. **9:16 вертикаль** — субъекты в центре кадра, минимум горизонтального действия.
7. **Kling pan/tilt инверсия** — всегда проверять терминологию перед записью промпта.
8. **Без текста и озвучки** — чисто визуальный сторителлинг с музыкой/звуком.
9. **Промпты для картинок и переходов — разные вещи:**
   - Картинки (Nano Banana/GPT): детальное описание статичной сцены (40-100 слов).
   - Переходы (Kling AI I2V): только движение и трансформация (20-40 слов).
